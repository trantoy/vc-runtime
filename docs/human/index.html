vc-runtime

A local-first realtime voice conversion runtime focused on latency, stability, provider transparency, and diagnostics.

Architecture Phase 0 Glossary Agent vision
Runtime first

Not a GUI wrapper

The UI is a client of the runtime. The runtime owns realtime audio, model execution, provider selection, and diagnostics.

Diagnostics first

Make latency visible

Every run should explain stage timing, queue growth, underruns, overruns, and actual provider usage.

RVC first

Narrow first target

The first model family is RVC through ONNX Runtime. Other model families wait until the runtime proves itself.
