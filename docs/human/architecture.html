Architecture

The runtime separates control plane from data plane so UI and configuration cannot stall realtime audio.

Home Phase 0 Glossary Technical source

Control plane

Config validation, model loading, provider selection, session lifecycle, UI, CLI, and metrics export.

Data plane

Audio callbacks, ring buffers, chunk scheduling, DSP, model worker deadlines, and output queue health.

Boundary rule

The audio callback never runs model inference, blocks on UI/control tasks, or performs heavy allocation.

First Implementation Target

input device
  -> CPAL callback
  -> ring buffer
  -> CPAL output callback
